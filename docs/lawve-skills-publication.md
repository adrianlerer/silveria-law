# Publicacion de skills Lawve

Esta regla separa skills publicables de skills reservadas.

## Principio

Publicar solo lo que ayude a la comunidad sin exponer prompts internos, estrategia propietaria, datos, umbrales, ruteos, ledgers ni mecanismos operativos sensibles.

## Taxonomia

| Tipo de skill | Publicable | Regla |
| --- | --- | --- |
| Skill pedagogica | Si | Puede explicar conceptos, verificacion, uso responsable y alfabetizacion. |
| Checklist profesional | Si | Puede publicarse si no contiene datos reales ni estrategia privada. |
| Plantilla normativa general | Si, con cautela | Debe aclarar jurisdiccion, fuentes y necesidad de revision profesional. |
| Patron de interfaz | Si | Puede publicarse como UX reusable si no revela logica propietaria. |
| Prompt interno | No | Reservado. No publicar prompts privados. |
| Routing, scoring o thresholds | No | Reservado. No publicar umbrales, rutas, rankings ni criterios sensibles. |
| Skill con datos/clientes | No | Reservado. No publicar casos reales, clientes, datasets privados ni corpus interno. |
| Skill hibrida | Parcial | Publicar estructura educativa; mantener privada la mecanica operativa. |

## Regla operativa corta

Publicamos skills educativas, checklists y patrones verificables. No publicamos prompts internos, estrategia propietaria, datos, umbrales, ruteos, ledgers ni mecanismos operativos sensibles.

## Ejemplos publicables

- Como verificar una cita antes de firmar.
- Semaforo de riesgo para uso de IA.
- Checklist de confidencialidad antes de subir documentos.
- Plantilla de adaptacion por jurisdiccion.
- Guia para explicar BYOK sin exponer API keys.

## Ejemplos no publicables

- Prompt exacto de evaluacion interna.
- Logica de scoring o confianza.
- Rutas de modelo o proveedor.
- Datos de clientes o ejemplos reales.
- Ledgers de costos, performance o decision.
- Estrategia de producto privada.

## Revision antes de publicar

1. Confirmar que no hay datos personales, clientes ni secretos.
2. Confirmar que no hay prompts internos ni rutas operativas.
3. Confirmar que la skill no permite reconstruir scoring, thresholds o estrategia.
4. Confirmar que la jurisdiccion y fuentes estan aclaradas.
5. Confirmar que la revision profesional queda obligatoria.
