# Open source responsable

SilverIA Law es open source como software, documentacion y recurso comunitario de accesibilidad legal.

No afirma ser:

- un modelo de IA abierto;
- un dataset abierto;
- un proveedor de IA;
- una exposicion de prompts o mecanicas privadas.

## Que esta abierto

- Codigo fuente de la PWA.
- Interfaz accesible y patrones de UX.
- Documentacion de arquitectura.
- Guias de IA, BYOK, API keys y modelos locales.
- Modulos pedagogicos para abogados senior.
- Patrones comunitarios como Radar BORA y radar internacional de boletines oficiales.
- Hoja de ruta local-first para asuntos, citas verificables y redlines revisables.

## Que no se publica

- API keys o credenciales.
- Datos reales de clientes, expedientes o usuarios.
- Prompts internos de productos privados.
- Ruteos, umbrales, scoring, ledgers o evaluaciones propietarias.
- Modelos, pesos, datasets o infraestructura de IA como si fueran propios.
- Estrategia comercial o know-how reservado de JustitIA, IntegridAI, CriptoIUS, Lawve u otros verticales.

## Grados de apertura en IA

En IA, open source no es binario. SilverIA Law usa estos terminos con precision:

- Software open source: codigo, licencia, documentacion y posibilidad de fork.
- Modelo fully open: arquitectura, pesos, entrenamiento, datos y documentacion completa.
- Open weights: pesos e inferencia, sin todo el dataset.
- Artefacto educativo abierto: guias, checklists, plantillas y patrones de capacitacion.

SilverIA Law entra en software open source y artefactos educativos abiertos. No entra en modelo fully open ni open weights.

## Regla de comunicacion

Texto recomendado:

> SilverIA Law es una PWA open source de accesibilidad legal y alfabetizacion IA. Su codigo y documentacion son publicos. No incluye modelos propios, datasets, API keys, prompts internos, datos privados ni mecanicas propietarias.

## Regla para contribuciones

Las contribuciones deben mantener la frontera publica/privada:

- no agregar secretos;
- no agregar datos reales;
- no copiar prompts internos;
- no publicar estrategias propietarias;
- no presentar soporte de IA real si la version es demo;
- no llamar "open model" a lo que solo es software abierto.

## Regla local-first

SilverIA Law puede publicar patrones generales de workspace legal local-first: asuntos separados, indices reconstruibles, citas verificables, redlines revisables y explicacion clara de proveedores IA.

No debe publicar mecanicas privadas de productos cerrados, prompts, routing, thresholds, scoring, ledgers, clientes, documentos reales ni estrategia propietaria.
