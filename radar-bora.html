Radar BORA: informe diario con enlaces oficiales y control humano

Radar BORA es una propuesta de modulo para que abogados, estudios y Colegios de Abogados reciban un resumen diario del Boletin Oficial de la Republica Argentina sin revisar manualmente cada seccion.

Idea central

  • Primero se capturan fuentes publicas.
  • Luego se filtra sin IA cuando sea posible.
  • La IA resume solo lo relevante.
  • La decision juridica queda en manos del profesional.

1. Estado actual

Esta pagina documenta el diseno del modulo. La PWA publica todavia no ejecuta un monitor real del Boletin Oficial, no llama a modelos de IA y no procesa documentos de usuarios. El objetivo es que cualquier institucion pueda entender como se implementaria sin ocultar costos ni credenciales.

2. Captura diaria

Un proceso programado consulta las secciones del BORA una vez por dia.

  • Primera Seccion: legislacion y avisos oficiales.
  • Segunda Seccion: sociedades y avisos judiciales.
  • Tercera Seccion: contrataciones.
  • Guarda fecha, seccion, rubro, titulo y enlace oficial.

3. Filtro barato antes de IA

No conviene que la IA lea todo si primero puede filtrarse por reglas simples.

  • Organismos: ARCA, BCRA, CNV, IGJ, ministerios.
  • Materias: laboral, fiscal, societario, consumo, datos personales.
  • Jurisdicciones, provincias, municipios o rubros.
  • Palabras clave elegidas por cada usuario o institucion.

4. IA solo para valor agregado

La IA debe trabajar sobre resultados ya filtrados.

  • Resumen en lenguaje claro.
  • Clasificacion de impacto probable.
  • Motivo por el cual podria importar.
  • Lista de enlaces oficiales para revisar.

5. Informe diario

El usuario recibe un informe breve y accionable.

  • Email diario o panel web.
  • Secciones por materia.
  • Alertas de alta prioridad.
  • Aviso visible: requiere revision profesional.

6. Costo para quien ofrece el servicio

El costo real depende de cantidad de usuarios, emails e IA usada. La regla economica es simple: capturar el BORA una sola vez por dia, filtrar con reglas baratas y usar IA solo sobre lo que quedo relevante.

Capa Costo esperado Como controlarlo
Captura BORA Bajo o casi cero Una ejecucion diaria, cache y deduplicacion.
Hosting Bajo al inicio Funciones programadas con limite y logs.
Email Depende de destinatarios Enviar un resumen diario, no multiples alertas innecesarias.
IA Variable Modelos chicos, prompts cortos, maximo de items por informe.

7. Suscripcion recomendada

Si el modulo se ofrece en serio, debe tener suscripcion o BYOK. No es sustentable prometer IA ilimitada gratis.

Plan Uso Limite sano
Comunidad Resumen general del dia para capacitacion. Sin personalizacion profunda.
Profesional Un abogado recibe perfiles de interes. Materias y palabras clave limitadas.
Estudio Varios usuarios y perfiles por practica. Cupo mensual de informes y alertas.
Colegio de Abogados Plan institucional para matriculados. Presupuesto cerrado, politicas y soporte.

8. Si usted usa ChatGPT: atencion

Una suscripcion de ChatGPT no es lo mismo que una API key. Para que una aplicacion use modelos de OpenAI necesita una clave de la plataforma de desarrolladores y facturacion API separada.

  1. Entrar a platform.openai.com/api-keys.
  2. Iniciar sesion con su cuenta de OpenAI.
  3. Crear una nueva clave secreta para este proyecto.
  4. Copiarla una sola vez y guardarla en un lugar seguro.
  5. No enviarla por WhatsApp, email, formularios publicos ni capturas de pantalla.

9. Como poner la API key sin exponerla

La clave debe quedar en el servidor o en variables de entorno del proveedor de hosting. Nunca debe quedar en HTML, CSS, JavaScript publico, GitHub, localStorage ni en una captura.

Paso 1: crear un backend, por ejemplo una API route o funcion serverless.
Paso 2: guardar la clave como variable de entorno llamada OPENAI_API_KEY.
Paso 3: la PWA llama a su backend, no directamente a OpenAI.
Paso 4: el backend aplica limites, registra uso y recien ahi llama al proveedor.
Paso 5: si se sospecha exposicion, revocar la clave y crear una nueva.

10. Ejemplo de arquitectura segura

PWA SilverIA Law
  -> /api/radar-bora/informe
    -> autenticar usuario
    -> verificar cupo de suscripcion
    -> leer cache del BORA
    -> filtrar por perfil
    -> llamar IA con OPENAI_API_KEY del servidor
    -> devolver informe con enlaces oficiales

11. Que debe decir el producto

Texto recomendado para evitar engaños:

Radar BORA genera un informe preliminar a partir de publicaciones oficiales y filtros configurados. Puede usar IA para resumir o clasificar, pero no reemplaza la revision del abogado ni constituye asesoramiento juridico automatizado.

12. Siguiente paso tecnico

  1. Crear backend separado para el modulo Radar BORA.
  2. Agregar cron diario para capturar secciones del BORA.
  3. Guardar cache estructurada por fecha, seccion y rubro.
  4. Crear perfiles de interes por usuario o institucion.
  5. Conectar IA con variable segura y limite mensual.
  6. Enviar email o mostrar panel con enlaces oficiales.
